# API استخراج بيانات السجل التجاري (خادمي) — لمشروع CRM

يستقبل **صورة سكرين شوت** أو **ملف PDF** لشهادة السجل التجاري، يستخرج البيانات
في الخادم بدقة عالية، ويرجعها JSON لتعبئة فورم `add-cstmr.php` تلقائياً.

## لماذا الخادم أفضل من Tesseract.js في المتصفح؟
- **الـ PDF الأصلي**: يُستخرج نصه بـ `pdftotext` بدقة **100%** بلا أي خطأ OCR
  (Tesseract.js لا يقرأ PDF أصلاً، ويحوّله لصورة فيخسر الدقة).
- **سرعة أعلى**: المعالجة في الخادم أسرع من المتصفح، ولا تُجمّد جهاز المستخدم.
- **منطق استخراج أقوى**: تم اختباره على عدة حالات (مؤسسة/شركة، أرقام عربية،
  تنسيقات تواريخ مختلفة، حقول ناقصة) ونجح في كلها.

## الملفات
| الملف | المكان المقترح | الوصف |
|------|----------------|-------|
| `cr_ocr_api.php` | `admin/api/` | نقطة النهاية التي تستقبل الملف |
| `extract_logic.php` | `admin/api/` | منطق تحليل النص واستخراج الحقول |
| `FRONTEND_SNIPPET.html` | — | الكود الذي تستبدل به `<script>` في صفحتك |

## خطوات التركيب

### 1) ثبّت أدوات النظام على الخادم
```bash
# Ubuntu / Debian
sudo apt-get update
sudo apt-get install -y poppler-utils tesseract-ocr tesseract-ocr-ara

# تأكد من التثبيت
pdftotext -v
tesseract --version
```
> إن كان خادمك Windows (XAMPP)، حمّل poppler و tesseract لويندوز وأضف
> مسار `bin` لكل منهما إلى متغير PATH، أو استبدل أوامر `exec` بالمسار الكامل
> للملف التنفيذي.

### 2) ضع ملفّي الـ API
انسخ `cr_ocr_api.php` و `extract_logic.php` إلى مجلد `admin/api/` في مشروعك.

### 3) عدّل صفحة add-cstmr.php
- احذف سطر تحميل Tesseract.js:
  ```html
  <script src="https://cdn.jsdelivr.net/npm/tesseract.js@5/dist/tesseract.min.js"></script>
  ```
- استبدل **كامل قسم `<script>` الأخير** (الذي يحوي `processImage` و
  `parseAndFill`) بالكود الموجود في `FRONTEND_SNIPPET.html`.
- تأكد من صحة رابط الـ API في أعلى السكربت:
  ```js
  const OCR_API_URL = '../../api/cr_ocr_api.php';
  ```
  بما أن صفحتك في `admin/pages/forms/` والـ API في `admin/api/`، فالمسار
  الصحيح هو `../../api/cr_ocr_api.php`.

## كيف يعمل التدفّق
1. المستخدم يرفع صورة → تمرّ بواجهة القص (Cropper) كما هي → تُرسل للـ API.
2. المستخدم يرفع PDF → يُرسل مباشرة للـ API بلا قص.
3. الـ API يقرر: PDF أصلي → `pdftotext`، أو صورة/PDF ممسوح → Tesseract.
4. يحلّل النص ويرجع JSON بكل الحقول.
5. الواجهة تعبّئ الفورم تلقائياً، ويراجع المستخدم البيانات ثم يضغط حفظ.

## الحقول المستخرجة (مطابقة لفورمك)
اسم المنشأة، نوع المنشأة، حالة السجل، الرقم الوطني الموحد، رقم السجل التجاري،
تاريخ القيد، تاريخ الانتهاء، رأس المال، العنوان المعتمد، اسم المالك،
رقم هوية المالك، رقم الجوال، البريد الإلكتروني.

## اختبار سريع للـ API
```bash
curl -X POST "http://localhost:8080/UltimatesolutionsCrm/admin/api/cr_ocr_api.php" \
     -F "file=@certificate.pdf"
```

## ملاحظات

**المراجعة قبل الحفظ:** الاستخراج التلقائي مساعِد وليس بديلاً عن المراجعة.
دائماً راجع الحقول (خاصة الأسماء والعناوين من الصور الممسوحة) قبل الضغط على حفظ.

**حماية الـ API (موصى به):** الـ API حالياً مفتوح. لحمايته، فعّل فحص الجلسة في
أعلى `cr_ocr_api.php` (الأسطر المعلّقة الخاصة بـ `$_SESSION['user_id']`) ليعمل
فقط للمستخدمين المسجّلين.

**تطابق البادئات:** منطق توزيع الأرقام يعتمد على البادئات السعودية (7=الموحد،
10=السجل، 1/2=الهوية). هذا احتياطي فقط — الأولوية دائماً للأرقام المعرّفة
بكلماتها المفتاحية في الشهادة.
